# 🐯 고려티켓 (KU Ticket) - 콘서트 좌석 예매 시스템

본 프로젝트는 고려대학교 영어영문학과 김지오 학생의 기말고사 대체 과제를 위해 제작된 **콘서트 좌석 예매 웹사이트**입니다. 
제출하신 PRD(기획서)의 11가지 핵심 기능을 완벽하게 구현하였으며, 포트폴리오 및 과제 채점자가 즉시 확인할 수 있도록 **GitHub Pages(정적 웹 호스팅)** 환경에 배포할 수 있는 형태로 빌드되었습니다.

특히 정적 서버 환경에서 백엔드 및 데이터베이스와의 연동을 증명하기 위해, 프론트엔드 액션에 반응하여 백엔드(Python) 코드 실행과 데이터베이스(SQLite3 SQL) 쿼리가 어떻게 흘러가는지 보여주는 **"실시간 DB 및 서버 로그 시뮬레이터 콘솔"**이 탑재되어 있습니다.

## 🚀 GitHub Pages 배포 방법 (과제 제출 가이드)

본 프로젝트는 추가적인 서버 세팅(Node.js, Python 등) 없이 브라우저에서 바로 동작하는 정적 웹앱입니다. 아래 단계에 따라 GitHub Pages에 배포하고 링크를 제출하세요.

1. **GitHub 계정 로그인 및 저장소 생성**
   - [GitHub](https://github.com)에 로그인한 후, 우측 상단의 **[New]** 버튼을 클릭하여 새 저장소(Repository)를 만듭니다.
   - 저장소 이름(Repository Name)을 입력합니다 (예: `concert-reservation`).
   - 저장소 공개 범위는 반드시 **Public**으로 설정합니다.

2. **프로젝트 파일 업로드**
   - 저장소가 생성되면 화면의 **"uploading an existing file"** 링크를 누릅니다.
   - 본 폴더에 있는 아래 3개 핵심 파일을 드래그 앤 드롭으로 올립니다.
     - `index.html`
     - `styles.css`
     - `app.js`
   - 하단의 **[Commit changes]**를 클릭해 저장합니다.

3. **GitHub Pages 활성화**
   - 저장소 상단 메뉴 중 **[Settings]** (톱니바퀴 아이콘)로 이동합니다.
   - 왼쪽 사이드바에서 **[Pages]** 메뉴를 클릭합니다.
   - **Build and deployment** 섹션의 **Branch** 설정을 `None`에서 `main` (또는 `master`)으로 변경하고, 우측의 폴더 위치는 `/ (root)`로 둔 뒤 **[Save]** 버튼을 누릅니다.

4. **웹사이트 접속 및 제출**
   - 약 1~2분 뒤 페이지를 새로고침하면 Settings -> Pages 탭 상단에 배포된 웹사이트 주소가 나타납니다.
   - 주소 형식: `https://[깃허브-아이디].github.io/[저장소-이름]/`
   - 이 URL 주소를 기말고사 과제 링크로 제출하시면 됩니다!

## 📋 PRD 핵심 기능 구현 대조표

| PRD 요구 기능 | 앱 내 구현 방식 | 시뮬레이터(DB 콘솔) 동작 |
| :--- | :--- | :--- |
| **기능 1. 회원가입** | 이름, 이메일, ID, PW 입력 폼 제공. 중복 아이디 검증 및 `localStorage` 가입 데이터 저장. | 아이디 중복 체크 시 `SELECT` 쿼리 실행, 가입 성공 시 `INSERT INTO users...` 쿼리 출력. |
| **기능 2. 로그인** | ID/PW 입력값 매칭 성공 시 세션 생성, 실패 시 오류 메시지 출력. ID/PW 찾기 모달 탑재. | 로그인 시 `SELECT` 회원 조회 쿼리 실행. 계정 찾기 시 Python `smtp_service` 발송 로그 출력. |
| **기능 3. 콘서트 검색** | 상단 실시간 검색창 필터링, 장르별 필터(POP, K-POP, Indie), 정렬(추천/날짜/잔여좌석). | 타이핑할 때마다 `SELECT * FROM concerts WHERE...` 조건절 쿼리가 실시간 갱신되어 출력. |
| **기능 4. 상세 정보 조회** | 콘서트 포스터 디자인 및 상세 일시, 러닝타임, 시네마틱 설명 제공. 실시간 잔여석 프로그레스 바. | 상세 진입 시 `SELECT * FROM concerts WHERE id = ?` 쿼리 실행. |
| **기능 5. 좌석 조회** | 10x10 격자 인터랙티브 좌석 배치도 구현. VIP(빨강), S(파랑), 스탠딩(초록) 등급별 컬러링. | 해당 공연의 이미 예약 완료된 좌석 정보를 `localStorage`에서 조회하여 배치도에 예약 차단 처리. |
| **기능 6. 좌석 선택** | 클릭 시 좌석이 활성화(Pulse 효과)되며 우측에 선택 좌석 칩 생성. 1인 최대 4석 예매 제한 탑재. | 선택/취소 시마다 프론트엔드 상태(State) 및 선택 정보 로그 출력. |
| **기능 7. 등급별 가격 계산** | VIP(15만원), S(12만원), 스탠딩(9만원) 가격 실시간 합산. **영문과 할인쿠폰(`KU_ENGL_2026`)** 10% 감면. | 할인 코드 적용 시 `SELECT * FROM coupons WHERE...` 검증 쿼리 실행 후 Python 할인 연산 수행. |
| **기능 8. 결제** | 신용카드(카드번호, 유효기간, CVC), 카카오페이 등 결제 수단 선택 모달 제공 및 승인 로딩 애니메이션. | 승인 진행 시 Python 결제 API(`payment_gateway.py`) 처리 로그 및 성공 트랜잭션 ID 발급 로그 출력. |
| **기능 9. 티켓 발급** | 고유 예매 번호 생성(`KU-2026-XXXX`) 및 e-티켓 상에 동적 QR 코드(SVG 벡터 그래픽) 실시간 출력. | 티켓 발급 후 `INSERT INTO bookings...` 및 Python e-티켓 이메일 발송 로그 출력. |
| **기능 10. 예매 내역 조회** | '내 예매 내역(My Tickets)' 탭에서 본인이 예매한 티켓의 공연명, 날짜, 좌석번호, 금액, 예매상태 실시간 조회. | 진입 시 `SELECT * FROM bookings WHERE user_id = ?` 쿼리 실행하여 UI에 목록 바인딩. |
| **기능 11. 예매 취소** | 마이페이지에서 취소 시 환불 재확인 컨펌창 출력 후 취소 완료 및 좌석 점유 상태 해제. | 취소 시 `UPDATE bookings SET status = 'cancelled'` 및 좌석 상태 복구 쿼리(`UPDATE seats...`) 실행. |

## 🎨 디자인 시스템 및 디테일

1. **글래스모피즘 (Glassmorphism)**
   - 미적인 충격을 위해 전체적으로 불투명한 뒷배경 블러(`backdrop-filter: blur(16px)`)와 반투명 보더 테두리를 적용하여 우주적이고 고급스러운 느낌을 주었습니다.
2. **반응형 인터페이스**
   - 모바일 환경(아이폰/갤럭시 세로화면)부터 데스크톱 크기까지 모든 그리드 및 좌석표가 화면을 벗어나지 않도록 스크롤과 플렉스박스 처리가 완료되었습니다.
3. **고려대학교 영어영문학과 특별 기믹**
   - 결제창 바로 앞의 쿠폰 입력란에 **`KU_ENGL_2026`**을 입력하여 쿠폰 적용을 클릭하면, **"영문과 특별 할인 (10%)"** 항목이 영수증에 추가되며 전 좌석 10% 차감 계산 혜택이 적용됩니다.
